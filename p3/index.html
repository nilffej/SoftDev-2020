Formerly known as Apache Cordova, PhoneGap is a framework that enables mobile app development using HTML, CSS, and Javascript.

Why use PhoneGap?

  • Uses commonly known languages
  • Creates cross-platform applications: Android, iOS, Windows
  • Offers a native app experience
  • It's capable of implementing native device tech (i.e. camera, contacts, push notifications)

PhoneGap is for you if...

  • you're looking to create simple mobile web apps.
  • you're looking to be time and cost efficient.
  • want an easily deployable product.

Disadvantages

  • iOS development requires iOS Developer membership
  • Unfit for gaming apps

There are two methods of interacting with PhoneGap on your desktop.
The Desktop App provides a more visual UI. The Command Line Interface is for those who prefer to work from the command line.

The mobile companion app is called PhoneGap Developer.
It is available in Google Play Store, but not in the App Store. The app allows you to load and debug from your mobile device and does not require a wired connection to your computer.

There are numerous templates to choose from to begin your PhoneGap project.
The command $ phonegap template list will return each template's name and its description.
In this walkthrough, we are starting a new blank project.
The command $ phonegap create myApp --template blank will create a new project with the template blank. Below is how the project may first appear in your IDE. The HTML file from which we will be working is located in the www/ directory.
The command $ phonegap serve starts a local server on your network to which you can connect to from a browser using the address returned in the terminal. A first launch may appear as shown below.

You can also establish a connection from the PhoneGap Developer app to demo your app on your mobile device.
To compile and officially launch your app, you must first visit the Adobe PhoneGap Build website and create an account. After logging, you will be redirected to a site that appears as below.

To upload your app to Adobe PhoneGap Build, you can either upload a zip folder of your project or connect your project via a GitHub HTTPS link. The advantage of using a GitHub connection is you can update your project on Adobe PhoneGap Build with a single button.
Your app can then be exported from the website as a downloadable and installable file. For Android devices, the file format will be an APK. Opening the APK file on an Android device will prompt the installation of the app.
